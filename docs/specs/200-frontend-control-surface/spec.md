# 200 Frontend Control Surface — Spec

> React/TS/Tailwind webview control surface for the Desktop Nano Cortex app. Owns the
> composition root (`App.tsx`, `main.tsx`), all MIDI feature components, hooks, service,
> and feature model files. Consumes IPC contracts owned by `210-frontend-ipc-contracts`.
> The app is a device-first Console workbench: outgoing PC/CC commands stay explicit,
> BLE device-state sync supplies live state where available, and any captured state
> without project verification stays provisional in diagnostics.

## Problem Statement

The Desktop Nano Cortex React frontend must be a fast desktop Console for the Nano Cortex. It
sends explicit Program Change / Control Change MIDI commands via the Tauri IPC bridge, syncs
live device state over BLE where available, and keeps unverified command families disabled or
clearly scoped to diagnostics.

Three distinct UI surfaces require honest-state discipline:

1. **Console workbench**: collapsible preset rail on the left, shared signal-path overview and
   six-knob device panel in the center, clickable footswitch deck below, and a right Utilities rail.
2. **Floating Tone Studio**: the dense tone surface for device-chain state, synced parameter
   values, and verified live writes; it opens from the shared signal path or Utilities rail.
3. **Diagnostics and About**: Advanced keeps diagnostics and optional Capture Lab tools; About carries support,
   warranty, source links, version, and a telemetry posture toggle.

The app uses an independent frontend codebase shaped around this product's live-control,
monitoring, and preset-editing workflow. The IPC surface it calls is documented in
`120-backend-ipc` and typed in `210-frontend-ipc-contracts`.

## Non-Goals

- Tauri IPC command/event wiring (owned by zone `120-backend-ipc`).
- IPC TypeScript contracts, shared providers (`useLogs`, `useTheme`), `LogPanel`, `ThemeToggle`, `ExperimentalBadge`, `EXPERIMENTAL_FEATURES`, and `featureFlags` (owned by zone `210-frontend-ipc-contracts`).
- BLE GATT transport, provisional Nano protocol decoding in Rust (owned by zone `110-backend-midi-ble`).
- Platform tray, global shortcuts, settings-store persistence (owned by zone `130-backend-platform`). Console document-level shortcuts are deferred for this release.
- `webmidi`, browser MIDI API, Web Bluetooth.
- Amp-model authoring, neural capture creation, capture/IR library management, or guided tone-generation workflows.
- Shipping unsupported BLE command families as confirmed before hardware graduation. FR-46 tracks the device-synced editor path and keeps unproven families disabled or diagnostic-only until `110 [NFR-8]` is satisfied per command family.
- SysEx editing or unverified device command families.
- Multi-device routing or virtual MIDI ports.
