# 120 Backend IPC — Spec

> Tauri IPC bridge. Owns all `#[tauri::command]` handlers, the `midi://*` event emitters,
> `AppState` (shared mutable device + BLE + NanoState), `AppError`, `BuildConfig`, and the
> serde-boundary mapping helpers. Calls into zones `100-backend-midi-usb` and
> `110-backend-midi-ble` for the actual I/O; does not own USB or BLE transport code directly.

## Problem Statement

The Rust backend must expose device I/O operations to the React/TS webview through a typed
Tauri IPC surface: commands (request/response) for actions, and pushed events for inbound
MIDI, connection lifecycle, and diagnostic logs. A shared `AppState` held across all
commands carries the live device, USB input listener handles, the BLE peripheral handle,
and the provisional `NanoState`.

`send_midi` must route identically structured byte arrays to either the USB or the BLE
transport depending on the transport kind stored in `AppState`, without the frontend needing
to know which physical transport is active. BLE send failure auto-disconnects and emits
`midi://disconnected` so the UI stays in sync without a separate poll.

The IPC layer is deliberately thin: it validates inputs, delegates to infrastructure, updates
`AppState`, and emits events. No MIDI protocol logic (port matching, packet decoding) lives here.

### Non-Functional Requirements

| ID    | Requirement                                                                                                                                                                           | Target                  |
| ----- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ----------------------- |
| NFR-1 | All `#[tauri::command]` handlers return `Result<T, String>` — never panic on user input.                                                                                              | Code invariant          |
| NFR-2 | `AppState` field access uses `tokio::sync::Mutex`; async commands hold locks for the minimum scope needed.                                                                            | Architectural invariant |
| NFR-3 | The `ble` feature gate is hermetic: removing the feature must produce a compilable binary with stub `ble_scan`/`ble_ping`/`get_ble_debug_log` returning `Err("BLE not compiled in")`. | CI-enforced             |
| NFR-4 | `midi://log` must not block the Tauri main thread; `append_protocol_log` opens and closes the file on each call (no persistent handle).                                               | Observed acceptable     |
| NFR-5 | The BLE disconnect watcher spawned inside `ble_scan` must not hold `AppState` locks across its 500 ms sleep.                                                                          | Architectural invariant |

## Acceptance Criteria

- [x] `list_ports` returns all USB MIDI output ports and emits a `midi://log` entry.
- [x] `connect` with a valid port name updates `AppState`, starts USB input listeners, and emits `midi://connected`.
- [x] `connect` with an invalid port name returns `Err` without modifying `AppState`.
- [x] `disconnect` stops USB listeners, drops BLE handle (feature-gated), emits `midi://disconnected`.
- [x] `send_midi` on a USB-connected device calls `connection::send_to_port`.
- [x] `send_midi` on a BLE-primary session routes via the Nano USB port when attached; otherwise errors without dropping the BLE session.
- [x] `send_midi` with a PC message (`0xC0xx` + `bytes[1] < 64`) updates `NanoState.active_preset_slot` and `bank`.
- [x] `get_state` returns `"connected"` when device is connected, `"disconnected"` otherwise.
- [x] `get_device_name` returns the connected device name or `None`.
- [x] `get_nano_state` returns the current `NanoState`.
- [x] `get_ble_capabilities` returns the current `CapabilityMatrix`.
- [x] `get_ble_debug_log` returns a packet log snapshot when `ble` is active; empty list when not.
- [x] `ble_scan` reuses an existing connected handle without rescanning.
- [x] `ble_scan` prevents concurrent scans via `ble_scanning` guard.
- [x] `ble_scan` spawns a disconnect watcher that calls `set_disconnected` and emits `midi://disconnected` on peripheral drop.
- [x] `ble_ping` returns `"BLE adapter OK"` when BLE adapter is available.
- [x] `trace_marker` rejects empty label or phase; writes `tracing::info!` and emits `midi://log`.
- [x] `export_settings_json` expands `~`, validates parent directory exists, writes file, returns absolute path.
- [x] `import_settings_json` rejects missing file, directory paths, and files > 10 MB.
- [x] `midi://log` is written to stdout and appended to `logs/protocol-lab.log` on every call.
- [ ] Full USB connect + send + disconnect smoke verified against a real Nano Cortex (hardware-only).
- [ ] BLE scan + send + watcher-detected disconnect verified against a real Nano Cortex (hardware-only).

## Non-Goals

- USB MIDI port enumeration / name matching (owned by zone `100-backend-midi-usb`).
- BLE GATT transport, packet decoding, and provisional sync (owned by zone `110-backend-midi-ble`).
- Platform tray, global shortcuts, and settings-store persistence (owned by zone `130-backend-platform`).
- Frontend TypeScript IPC contracts and React hooks (owned by zone `210-frontend-ipc-contracts`).
- MIDI routing, virtual ports, or multi-device scenarios.
- SysEx editing or unverified Nano Cortex preset command families.

## Dependencies

- `tauri = "2"` — command handler macro, `AppHandle`, `Emitter`, state injection.
- `tokio` — `async` command support, `Mutex`, `spawn`, `time::timeout`.
- `serde` + `serde_json` — command/event payload serialization.
- `midir` — `MidiInputConnection<()>` held in `AppState.midi_input_connections`.
- `tracing` — structured logging from `trace_marker` and `emit_log`.
- **Zone 100**: `port_manager::list_output_ports / list_input_ports`, `listener::start_listener`, `connection::send_to_port`, `MidiPort`, `MidiMessage`.
- **Zone 110**: `ble::find_and_connect_with_log`, `ble::get_adapter`, `BleHandle`, `BlePacketLogEntry` (all behind `ble` feature).
- **Zone 130**: consumes `AppState` reference; tray/shortcuts setup is not done here.
